## Gerenciamento de Configuração e Versão

Durante o desenvolvimento de um software, é necessário colaborar com diversas equipes. O grau de colaboração das equipes, inclusive, entra como fator decisivo para o sucesso de um projeto de desenvolvimento de software, como pode ser observado nos tipos de riscos organizacionais que podem impactar negativamente a evolução de um projeto (SOMMERVILLE, 2019).

Durante as diferentes etapas de desenvolvimento de software, temos grupos diferentes que são responsáveis pelo gerenciamento de configuração e versão. Gerenciar corretamente as configurações e as versões contribui para o correto funcionamento do software, além de facilitar a correção de erros. 

Na fase de desenvolvimento, a equipe responsável pelo desenvolvimento é responsável pelo gerenciamento de versões e configurações (SOMMERVILLE, 2019). No decorrer do desenvolvimento, a boa prática é utilizar alguma ferramenta de versionamento para que novas funcionalidades e correção de erros sejam controladas em diferentes versões do software. Com o auxílio de testes de software, é possível identificar erros e fazer com que uma versão antiga seja restabelecida, por exemplo.

Gerenciar versões no ambiente local de desenvolvimento, mesmo ao se trabalhar de forma individual, é uma boa prática. Ao tentar implementar novas funcionalidades, em caso de erros é possível voltar com alguma versão anterior funcional. O gerenciamento de versões usando uma ferramenta como o Git se torna ainda mais essencial quando trabalhamos em equipe, pois é uma maneira eficiente de favorecer a colaboração, isto é, tornar possível que mais de uma pessoa efetue modificações em um código e as versões diferentes possam ser integradas de forma coerente.

De acordo com sua especificação de requisitos, um software funciona de acordo com algumas premissas. Estas premissas podem estar relacionadas a configurações próprias relacionadas a aspectos internos do sistema, ou então a configurações relacionadas a sistemas externos e outras condições do ambiente de implantação do sistema. Assim como sensores de temperatura precisam de calibração, o mesmo ocorre em softwares para efetuar integrações de forma correta. Uma má configuração de uma taxa de comunicação entre sistemas diferentes pode impossibilitar sua troca de informações, por exemplo.

Na fase de testes do sistema, nenhuma funcionalidade adicional é integrada e o foco no gerenciamento de versões e configurações está em identificar e corrigir erros, além de lidar com vulnerabilidades de segurança da informação (SOMMERVILLE, 2019). Nesta fase, o gerenciamento de configurações está sob responsabilidade de uma equipe de gestão de qualidade.

Já na fase de lançamento, temos versões de lançamento diferentes para utilização livre por usuários. Nesta fase, usualmente uma equipe de manutenção é responsável pela sustentação do sistema, gerenciando suas configurações e versões. Nesta fase, solicitações de mudanças por usuários e correções de defeitos são realizadas (SOMMERVILLE, 2019).

Docker é uma ferramenta interessante para facilitar a implantação de sistemas. Com o uso de Docker, temos imagens Docker de sistemas operacionais com diversas dependências instaladas, de forma que o problema de que “funciona bem só em uma máquina específica de quem desenvolveu” é mitigado. Ao invés de precisar replicar manualmente todas as configurações de um ambiente utilizado para desenvolvimento de um sistema, ao se usar Docker a ideia é obter uma imagem Docker específica e usá-la diretamente.

A ideia principal do gerenciamento de versões nas diferentes fases de desenvolvimento, testes e lançamento, é mitigar defeitos em ambiente de produção. Gerenciar corretamente as versões pode ser um fator decisivo para uma empresa lidar bem em ocasiões de falhas e erros, de forma a diminuir a ocorrência de incidentes em ambiente de produção.

  

  

**Atividade Extra**

Obtenha algum código que já tenha desenvolvido em qualquer linguagem de programação. Refaça o código, mas usando alguma ferramenta de gerenciamento de versão como o GitHub, realizando commits com comentários. Ao final do processo, você pode obter um código muito similar ao original, porém com o histórico de versões que demonstra como o código evoluiu.
